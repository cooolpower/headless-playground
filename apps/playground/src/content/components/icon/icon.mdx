# Icon

Icon 컴포넌트는 애플리케이션 전반에 걸쳐 일관된 아이콘 표시를 제공하는 Headless 컴포넌트입니다. [Lucide Icons](https://lucide.dev/icons/)를 기본 아이콘 라이브러리로 사용합니다.

## 기본 사용법

Icon 컴포넌트는 Lucide 아이콘을 래핑하여 일관된 크기와 스타일을 제공합니다.

import { Playground } from '@/components/playground/playground';
import { CssSnippet } from '@/components/playground/css-snippet';
import { IconCss } from '@repo/ui';
import {
  DemoIconProvider,
  DemoIcon,
  IconControls,
  IconGallery,
} from './icon.demo';

<DemoIconProvider>
  <Playground preview={<DemoIcon />} controls={<IconControls />} />
</DemoIconProvider>

```tsx
import { Icon } from '@repo/ui';
import { Home, Search, Heart } from 'lucide-react';

export function MyComponent() {
  return (
    <>
      <Icon icon={Home} size="medium" />
      <Icon icon={Search} size="medium" />
      <Icon icon={Heart} size="medium" />
    </>
  );
}
```

또는 children을 사용하여 커스텀 아이콘을 표시할 수 있습니다:

```tsx
<Icon size="medium">
  <span>🏠</span>
</Icon>
```

## Size

Icon은 다양한 크기를 지원합니다.

```tsx
import { Home } from 'lucide-react';

<Icon icon={Home} size="small" />
<Icon icon={Home} size="medium" />
<Icon icon={Home} size="large" />
```

## API

### Icon Props

| Property     | Description                    | Type                             | Default    |
| ------------ | ------------------------------ | -------------------------------- | ---------- |
| icon         | Lucide 아이콘 컴포넌트         | `LucideIcon`                     | -          |
| size         | 아이콘 크기                    | `'small' \| 'medium' \| 'large'` | `'medium'` |
| className    | 추가 CSS 클래스                | `string`                         | -          |
| injectStyles | 기본 스타일 주입 여부          | `boolean`                        | `true`     |
| children     | 아이콘 콘텐츠 (SVG, 이모지 등) | `React.ReactNode`                | -          |

## 특징

- **Lucide 통합**: [Lucide Icons](https://lucide.dev/icons/) 라이브러리와 완벽하게 통합
- **일관된 크기**: size prop을 통해 일관된 아이콘 크기 제공
- **유연한 콘텐츠**: Lucide 아이콘, SVG, 이모지, 텍스트 등 다양한 형태의 아이콘 지원
- **스타일 커스터마이징**: className을 통해 추가 스타일 적용 가능
- **트리 쉐이킹**: 필요한 아이콘만 번들에 포함되어 최적화된 성능

## 왜 Lucide를 사용하나요?

Headless 컴포넌트 시스템을 구축하는 현재 상황에서 Lucide를 계속 사용하는 것을 권장합니다:

1. **이미 설치됨**: 프로젝트에 이미 설치되어 있고 input-number에서 사용 중
2. **풍부한 아이콘**: 1000개 이상의 고품질 아이콘 제공
3. **일관된 디자인**: 모든 아이콘이 동일한 디자인 시스템을 따름
4. **트리 쉐이킹**: 필요한 아이콘만 번들에 포함되어 번들 크기 최적화
5. **TypeScript 지원**: 완벽한 타입 안정성 제공
6. **활발한 커뮤니티**: 지속적인 업데이트와 유지보수
7. **디자이너 불필요**: 디자이너 없이도 충분히 많은 아이콘 사용 가능

자체 아이콘 시스템을 만드는 것은:

- 시간과 리소스가 많이 소요됨
- 디자이너가 필요한 경우가 많음
- 유지보수 부담이 큼
- 일관성 유지가 어려움

따라서 Lucide를 계속 사용하는 것이 가장 효율적입니다.
